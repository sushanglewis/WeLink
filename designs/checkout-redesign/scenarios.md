# 用户场景：Lincoln TUI 录音工具

**会话 ID**: 2026-06-28-stakeholder-checkout  
**设计 ID**: checkout-redesign

---

## 目标用户

| 用户 | 描述 | 核心诉求 |
|---|---|---|
| 产品经理（PM） | 使用 Lincoln 工作流进行需求访谈 | 快速、可见地启动/停止录音 |
| 技术负责人 | 维护 Lincoln 工作流工具链 | 工具统一、产物兼容、低维护成本 |

## 主场景

### 场景 1：在 Conductor 工作区开始访谈录音

1. PM 在 Conductor 中打开工作区终端。
2. PM 输入 `lincoln` 并回车。
3. TUI 启动，自动检测当前 git 分支与日期，生成 `session_id`。
4. TUI 显示“正在录音”动态界面，时长开始计时。
5. PM 与利益相关者进行访谈。
6. 访谈结束后，PM 按 `Enter` 停止录音。
7. TUI 提示“是否立即处理访谈？”，PM 选择“是”。
8. 产物保存，并触发 `claude process-interview`。

**来源**: `[00:00:02 - 00:00:37]`

## 边界场景

### 场景 2：误触取消

- PM 录音过程中误按 `q` 或 `Ctrl+C`。
- 工具优雅取消，不保存录音，不触发后续处理。
- TUI 显示“已取消”，并退出到普通终端。

### 场景 3：无 git 分支的工作区

- 当前目录不是 git 仓库。
- 工具使用日期生成 `session_id`，分支字段留空或使用默认值。

### 场景 4：非交互式环境（CI/脚本）

- `lincoln` 检测到非 TTY 环境。
- 降级为普通 CLI 输出，或使用 `--no-tui` 模式（MVP 外）。

## 非目标场景

- 在浏览器中点击按钮录音。
- 录音过程中实时显示转写文字。
- 多轨音频录制与后期编辑。
- 跨设备云端同步录音文件。
