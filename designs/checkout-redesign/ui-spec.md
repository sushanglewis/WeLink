# UI 规格：Lincoln TUI 录音工具

**会话 ID**: 2026-06-28-stakeholder-checkout  
**设计 ID**: checkout-redesign

---

## 1. 目标用户流

1. PM 在终端输入 `lincoln`。
2. TUI 启动并立即开始录音。
3. PM 查看“正在录音”动态状态（脉冲红点、时长、音频电平条）。
4. 访谈结束后，PM 按 `Enter` 停止录音。
5. TUI 询问是否立即处理访谈。
6. PM 选择后，TUI 退出并返回普通终端。

## 2. 界面列表

### 2.1 录音主界面（Recording Screen）

**布局**：

```
┌─────────────────────────────────────────┐
│ Lincoln Recorder                        │
│ Session: 2026-06-28-stakeholder-checkout│
│ Topic: 结算流程 redesign 需求访谈       │
│ Branch: pm-workflow-integration-plugin  │
│                                         │
│        🔴 正在录音                       │
│        00:01:23                         │
│                                         │
│  Audio Level: ████████░░ 78%            │
│                                         │
│  按 Enter 停止 · 按 q 取消              │
└─────────────────────────────────────────┘
```

**元素**：

| 元素 | 类型 | 说明 |
|---|---|---|
| 标题 | Text | "Lincoln Recorder" |
| 会话信息 | Text | session_id / topic / branch |
| 录音状态 | Text + 脉冲红点 | "正在录音"，红点每秒脉冲一次 |
| 时长 | Text | `HH:MM:SS` 格式 |
| 音频电平条 | 进度条 | 根据 `amplitude` 实时更新 |
| 操作提示 | Text | "按 Enter 停止 · 按 q 取消" |

**状态**：

- **默认**：正常录音中，红点脉冲，时长递增，电平条随音频变化。
- **低音量**：电平条低于 10% 时显示浅灰色提示（可选）。
- **错误**：录音失败时，红点变 ❌，显示 `error_message`。

### 2.2 停止确认界面（Stop Confirmation）

**布局**：

```
┌─────────────────────────────────────────┐
│ 录音已保存                              │
│ 时长：00:01:23                          │
│ 文件：recordings/...m4a                 │
│                                         │
│ 是否立即处理访谈？                      │
│ [是] 触发 claude process-interview      │
│ [否] 仅保存，稍后手动处理               │
└─────────────────────────────────────────┘
```

**交互**：

- 上下箭头选择，Enter 确认。
- 或按 `y` 直接选择“是”，按 `n` 直接选择“否”。

### 2.3 取消界面（Cancelled Screen）

**布局**：

```
┌─────────────────────────────────────────┐
│ 录音已取消                              │
│ 未保存任何文件                          │
└─────────────────────────────────────────┘
```

**行为**：

- 显示 1 秒后自动退出 TUI。
- 或按任意键立即退出。

## 3. 交互说明

| 按键 | 状态 | 行为 |
|---|---|---|
| `Enter` | 录音中 | 停止录音，进入停止确认界面 |
| `q` | 录音中 | 取消录音，进入取消界面 |
| `Ctrl+C` | 任意 | 优雅取消，清理临时文件，退出 TUI |
| `y` | 停止确认 | 触发 `claude process-interview` |
| `n` | 停止确认 | 仅保存，退出 TUI |
| `↑/↓` | 停止确认 | 切换选项 |

## 4. 组件状态

### 录音状态指示器

| 状态 | 视觉 |
|---|---|
| 录音中 | 红色脉冲圆点 + "正在录音" |
| 停止 | 绿色圆点 + "录音已保存" |
| 取消 | 灰色圆点 + "录音已取消" |
| 错误 | 红色 ❌ + 错误信息 |

### 音频电平条

| 电平 | 颜色 |
|---|---|
| 0-30% | 绿色 |
| 31-70% | 黄色 |
| 71-100% | 红色 |

## 5. 空/加载/错误状态

- **加载**：TUI 启动后 1 秒内显示 "正在初始化麦克风..."，然后开始录音。
- **空音频**：电平条长时间为 0 时，显示提示 "未检测到音频输入，请检查麦克风"。
- **错误**：ffmpeg 未安装、麦克风无权限、磁盘空间不足等，显示具体错误并退出。

## 6. 无障碍说明

- TUI 使用高对比度颜色，确保在浅色/深色终端主题下可读。
- 所有操作均提供键盘入口，无需鼠标。
- 关键状态变化（开始/停止/取消）均有明确的文字提示。

## 7. 实现约束

- 使用 `ink` + React 19 + TypeScript。
- 调用现有 Python `record-interview` CLI 作为录音后端。
- 音频电平通过 ffmpeg 实时分析或 Node.js 音频流获取。
- 全局配置 `~/.lincolnrc` 使用 YAML 格式。
