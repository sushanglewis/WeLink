# Scenarios: 企业 AI 工作台一级落地页

## 目标用户

### 主要用户：组织内部普通员工
- 每天登录 WeLink 处理工作
- 对 AI 工具有基础认知，但希望企业有一个统一的 AI 入口
- 希望通过自然语言快速完成办公任务

### 次要用户：企业/部门管理员
- 负责查看使用数据以评估 AI 落地效果
- 后续规划中会参与 skill 配置（本期不做）

### 间接受益者：业务负责人
- 关注 AI 投入是否带来实际业务价值
- 通过数据挖掘发现新的 AI 使用场景

## 主场景

### 场景 1：员工打开 AI 工作台

**触发**：员工登录 WeLink，点击左侧导航"AI 工作台"。  
**流程**：
1. WeLink 加载 AI 工作台壳层。
2. 壳层向企业后端请求 LibreChat 访问 token。
3. 后端校验 IM 登录态并返回 token + LibreChat URL。
4. 壳层通过 iframe 加载 LibreChat 指定路由。
5. 员工看到 LibreChat 原生界面。

**预期结果**：员工在 WeLink 内无感进入 LibreChat，无需二次登录。

### 场景 2：员工通过自然语言查询知识

**触发**：员工在 LibreChat 输入框输入"请解释一下我们的报销流程"。  
**流程**：
1. LibreChat 识别意图为知识库问答。
2. 调用 LibreChat RAG API，使用企业 embed 模型检索内部文档。
3. 返回答案并展示引用来源。

**预期结果**：员工获得基于企业知识的准确答案。

### 场景 3：员工使用 Canvas 生成文档

**触发**：员工在 LibreChat 中请求"帮我写一份项目周报"。  
**流程**：
1. LibreChat 进入 Artifacts 模式。
2. 生成 Markdown/HTML 格式的项目周报。
3. 员工可在线编辑并下载。

**预期结果**：员工获得可下载、可编辑的文档内容。

### 场景 4：员工与企业 model config 对话

**触发**：员工向 LibreChat 提问通用问题。  
**流程**：
1. LibreChat 调用企业统一 model config。
2. 返回生成式回答。

**预期结果**：员工使用的是企业管控的模型算力。

## 边界场景

| 场景 | 处理方式 |
|------|---------|
| 员工提问超出企业知识库范围 | LibreChat 按通用模型回答，或提示"未找到相关企业知识" |
| 企业 model config 不可用 | 展示降级提示，引导员工稍后重试 |
| LibreChat 服务异常 | WeLink 展示兜底页面或友好错误提示 |
| token 过期 | 壳层自动向后端申请新 token 并刷新 iframe |

## 非目标场景

- 员工在 AI 工作台内使用企业自定义 skill/Agent（本期不做，列入规划）。
- 员工通过 AI 工作台处理督办事项（依赖 `designs/supervision-collab-va` 上线）。
- 员工管理其他员工的权限或 skill 订阅（本期不做）。
- 员工使用 AI 工作台访问外部互联网大模型（统一走企业 model config）。
