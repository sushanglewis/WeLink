# 用户故事：Lincoln TUI 录音工具

**会话 ID**: 2026-06-28-stakeholder-checkout

---

## 故事 1：作为 PM，我需要在终端中一键开始录音

**作为** 使用 Lincoln 工作流的产品经理，  
**我希望** 在 Conductor / AI 工具工作区的终端中输入 `lincoln` 即可启动录音，  
**从而** 不必记忆复杂的 `record-interview` 命令和参数。

**来源**: `[00:00:02 - 00:00:19]`

**验收标准**:
- 全局安装后，`lincoln` 命令在 PATH 中可用。
- 命令在当前工作区启动，自动派生 `session_id`。
- 无需手动指定 `--branch`、`--design-id` 等参数（使用默认值或自动检测）。

## 故事 2：作为 PM，我需要在录音时看到清晰的 TUI 状态

**作为** 正在录音的 PM，  
**我希望** 终端中显示一个带有动态效果的“正在录音”TUI 界面，  
**从而** 确认录音正常进行，并随时了解已录音时长。

**来源**: `[00:00:19 - 00:00:26]`, `[00:00:37 - 00:00:52]`

**验收标准**:
- TUI 显示脉冲/动态效果表示录音中。
- TUI 显示当前录音时长（如 `00:01:23`）。
- TUI 提供明确的停止指引（如“按 Enter 停止”）。

---

## 故事 3：作为 PM，我需要在停止录音后无缝衔接现有工作流

**作为** 完成访谈的 PM，  
**我希望** 停止录音后产物自动保存到标准位置，并可选择立即触发 `process-interview`，  
**从而** 保持 Lincoln 现有工作流不变。

**来源**: `[00:00:37 - 00:00:52]`

**验收标准**:
- 录音文件保存到 `recordings/<session-id>.m4a`。
- 元数据写入 `interviews/<session-id>/metadata.json`。
- 停止后询问是否触发 `claude process-interview`。

## 故事 4：作为 PM，我希望工具跨 AI 工具通用

**作为** 同时使用 Claude、Codex、OpenCode 等多种 AI 工具的 PM，  
**我希望** 录音工具是独立的终端 TUI，不绑定任何单一 AI 工具，  
**从而** 无论在哪个工具的工作区都能一致使用。

**来源**: 用户澄清反馈

**验收标准**:
- 工具以 npm 全局包形式发布。
- 不依赖 Claude / Codex / OpenCode 的私有 API 或插件机制。
- 只要工作区有终端即可使用。
