# 可行性分析: issue-12

## 业务可行性

### 市场机会

- 飞书 Aily 和钉钉悟空验证了“AI + 协同办公”的市场需求。
- WeLink 已有数字员工和 Teable 基础，不需要从零教育市场。
- 短期补齐人-机协同系统，可直接提升现有产品价值，为长期路线保留选项。

### 竞争优势

| 优势 | 说明 |
|------|------|
| 已有数字员工基础 | 不需要重新构建 AI 入口。 |
| Teable 中介平台 | 已有人-机协作的载体，可扩展为统一协作层。 |
| 开源方案成熟 | 协作文档、日程、会议、知识库均有成熟开源方案可选。 |
| 灵活路线 | 短期不绑定独立 AI 工作台，降低决策风险。 |

### 业务风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户心智不统一 | 数字员工、Teable、部门 BOT 入口分散 | 统一品牌、统一入口、统一交互范式。 |
| 开源方案 License 风险 | AGPL/BSL 等可能影响商业分发 | 优先选择 MIT/Apache 2.0；必要时购买商业授权。 |
| 竞品迭代快 | 飞书/钉钉持续投入 AI | 聚焦差异化场景（组织嵌入、Teable 中介），不追全功能。 |

## 技术可行性

### 推荐技术栈

| 能力 | 推荐方案 | 技术栈 | 集成方式 |
|------|----------|--------|----------|
| 企业知识库 | BookStack 或 Wiki.js | PHP/Laravel 或 Node.js/Vue | API + SSO |
| 技能分发/Agent 平台 | Dify（内部）或 Coze Studio（分发） | Python/React 或 Go | API + SDK |
| 协作文档 | OnlyOffice Document Server | Node.js/C++ | WOPI + iframe |
| 协作日程 | Nextcloud Calendar 或 Baikal | PHP/CalDAV | CalDAV + iframe |
| 协作会议 | Jitsi Meet | TypeScript/WebRTC | iframe + SDK |

### 与现有架构的兼容性

- **数字员工 Runtime**: 现有 Mattermost BOT + skills + MCP/CLI 架构可继续复用。
- **Teable**: 作为中介平台，需要扩展 API 以支持文档、日程、会议的嵌入和状态同步。
- **权限体系**: 需要把 Mattermost/Baserow/Teable 的权限统一到数字员工访问控制层。

### 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 多系统集成复杂 | 延迟增加、数据不一致 | 引入统一 API 网关和事件总线。 |
| 中文搜索/分词体验差 | 知识库检索效果差 | 选择支持中文分词的方案，或接入国产 embedding 模型。 |
| 大规模会议稳定性 | Jitsi 千人会议需调优 | 初期限制会议规模，逐步扩展基础设施。 |
| Skill 安全隔离 | 恶意 skill 可能越权 | 建立 skill 沙箱、权限声明、人工确认机制。 |

## 法律/合规可行性

### License 评估

| 项目 | License | 嵌入/分发建议 |
|------|---------|---------------|
| BookStack | MIT | 可自由嵌入和商业分发。 |
| Wiki.js | AGPL | 作为独立服务调用，避免代码链接。 |
| OnlyOffice | AGPL / 商业版 | 采购商业版以获得嵌入授权。 |
| Nextcloud | AGPL | 作为独立服务调用。 |
| Baikal | AGPL | 作为后端服务调用。 |
| Jitsi Meet | Apache 2.0 | 可自由嵌入和商业分发。 |
| Dify | Apache 2.0 + 附加条款 | 内部使用风险低；白标/分发需商业授权。 |
| Coze Studio | Apache 2.0 | 可自由二次开发和商业分发。 |

### 国产化/数据合规

- 所有推荐方案均可私有化部署在国产服务器/操作系统上。
- 数据不出域，满足数据安全法和个人信息保护要求。
- LLM 可接入国产模型（通义千问、文心一言、DeepSeek 等）。

## 实施可行性

### 推荐实施顺序

| 阶段 | 时间 | 目标 |
|------|------|------|
| Phase 1 | 1-2 个月 | 企业知识库上线（BookStack 或 Wiki.js）；数字员工可基于知识库回答。 |
| Phase 2 | 2-3 个月 | Skill 市场 v1 上线；支持官方 skills 的安装和分配。 |
| Phase 3 | 3-4 个月 | 协作文档集成（OnlyOffice）；数字员工可生成文档。 |
| Phase 4 | 4-6 个月 | 协作日程、协作会议集成。 |
| Phase 5 | 6-12 个月 | 评估多 Agent 编排需求，决定是否引入开发者后台。 |

### 资源估算

| 能力领域 | 前端 | 后端 | 设计 | 测试 | 总计人月 |
|----------|------|------|------|------|----------|
| 企业知识库集成 | 1 | 1 | 0.5 | 0.5 | 3 |
| 技能分发 | 1.5 | 2 | 0.5 | 1 | 5 |
| 协作文档集成 | 1 | 1 | 0.5 | 0.5 | 3 |
| 协作日程集成 | 1 | 1 | 0.5 | 0.5 | 3 |
| 协作会议集成 | 1 | 1 | 0.5 | 0.5 | 3 |
| 统一权限/网关 | 0.5 | 2 | 0 | 0.5 | 3 |
| **合计** | | | | | **20 人月** |

## 推荐决策

1. **短期（3-6 个月）**：按 P0 → P1 顺序补齐人-机协同系统，不走独立 AI 工作台路线。
2. **开源选型**：
   - 知识库：BookStack（MIT，低风险）或 Wiki.js（功能强，需评估 AGPL）。
   - 技能分发：Dify（功能强，内部使用）或 Coze Studio（License 干净，长期可分发）。
   - 协作文档：OnlyOffice（采购商业版）。
   - 日程：Nextcloud Calendar 或 Baikal。
   - 会议：Jitsi Meet。
3. **长期（6-12 个月）**：保留 AI 工作台可能性，但以“开发者后台/高级模式”延迟引入，不对普通用户暴露。
