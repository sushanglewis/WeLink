# 给 Office 套件厂商的需求描述：人-机协作文档能力

**背景项目**：WeLink Issue #12 — IM中的AI应用  
**目标厂商**：国内多人协作 Office 套件（文档/表格/幻灯片）  
**日期**：2026-07-20  

---

## 1. 我们的产品背景

我们已经实现了一套类似飞书 Aily 的 AI 协同办公能力：

- **IM 即时通信**：基于 Mattermost 的企业内部即时通讯。
- **组织架构机器人（数字员工）**：每个组织成员可以在 IM 中直接对话的 BOT。BOT 拥有组织内的独立身份（岗位、权限、技能约束），通过自然语言与用户交互。
- **Skills / CLI / MCP 编排**：BOT 的能力通过 skills 描述，skills 可以通过 CLI 或 MCP 协议调用外部系统（如 Teable/Baserow）。
- **人-机协作中介平台 Teable**：人与 AI 在任务、状态、结果交付上的协作载体。

目前我们**缺少在线协作文档能力**（Word、Excel、PPT）。因此希望通过与贵司 Office 套件集成，让 BOT 具备“读写在线 Office 文档”的能力，从而扩展人-机协作场景。

---

## 2. 我们的目标

让嵌入组织架构的 BOT 能够：

1. **被用户通过自然语言调用**，在对话中直接操作在线 Office 文档。
2. **独立完成文档类任务**，如创建周报、填写表格、生成 PPT、总结会议纪要。
3. **与人协作完成复杂文档**，AI 生成初稿，人工在 Office 套件中审阅、修改、确认。
4. **通过 BOT 独立 token 鉴权**，确保 AI 的操作权限与调用者权限一致，且可追溯。

整体架构对标：**飞书 Aily + 飞书文档/多维表格/幻灯片 + lark-cli**。

---

## 3. 期望的协作场景

### 场景 1：自然语言创建文档

> 用户在 IM 中 @周报助手：
> “帮我写一份本周销售周报，汇总 Teable 中的客户跟进记录。”

- BOT 识别意图，调用“文档创建 skill”。
- BOT 通过 CLI/API 读取 Teable 数据。
- BOT 在 Office 套件中创建一份文档初稿。
- BOT 将文档链接返回给用户，用户在 Office 中继续编辑。

### 场景 2：表格数据自动填充

> 用户 @数据助手：
> “把昨天销售群里提到的订单信息填到《7月订单表》里。”

- BOT 解析群聊消息，提取结构化信息。
- BOT 调用“表格写入 skill”，定位到对应表格和行。
- BOT 将数据写入表格，并标记来源。
- 用户收到确认消息，可在 Office 中查看和修正。

### 场景 3：PPT 自动生成

> 用户 @汇报助手：
> “基于《Q3 产品规划》文档，生成一份给管理层的汇报 PPT。”

- BOT 读取源文档内容。
- BOT 调用“幻灯片生成 skill”，按模板生成 PPT。
- BOT 返回 PPT 预览链接，用户可调整版式和内容。

### 场景 4：文档审阅与评论回复

> 用户 @审阅助手：
> “帮我看看这份方案文档有没有遗漏。”

- BOT 读取文档内容。
- BOT 基于知识和规则生成审阅意见。
- BOT 以评论形式写入文档，用户逐条确认。

### 场景 5：会议后自动沉淀

> 会议结束后，BOT 自动：
> 1. 读取会议录音/纪要；
> 2. 生成会议纪要文档；
> 3. 提取行动项写入任务表；
> 4. 将会议文档归档到知识库。

## 4. 对 Office 套件厂商的具体要求

### 4.1 文档/表格/幻灯片 API 能力

贵司 Office 套件需要提供完整的 Open API，支持 BOT 通过程序化的方式完成以下操作：

| 能力 | 文档 | 表格 | 幻灯片 |
|------|------|------|--------|
| 创建 | ✅ 按模板/空白创建 | ✅ 创建并写入表头 | ✅ 按模板创建 |
| 读取 | ✅ 读取全文、段落、表格 | ✅ 读取单元格/区域/全部 | ✅ 读取幻灯片内容 |
| 写入 | ✅ 追加、替换、插入块 | ✅ 写入单元格/行/列 | ✅ 写入页面、替换内容 |
| 更新 | ✅ 修改指定段落 | ✅ 更新单元格 | ✅ 更新页面 |
| 评论 | ✅ 添加/回复评论 | ✅ 添加评论 | ✅ 添加评论 |
| 权限 | ✅ 获取/设置文档权限 | ✅ 获取/设置表格权限 | ✅ 获取/设置权限 |
| 格式 | ✅ 支持标题、列表、表格、图片 | ✅ 支持公式、数据透视 | ✅ 支持图表、布局 |

**参考对标**：飞书开放平台文档 API、多维表格 API、幻灯片 API；lark-cli 的 `docs`、`base`、`sheets`、`slides` 能力。

### 4.2 CLI / MCP 支持

为了让 BOT 的 skills 能够方便地调用 Office 能力，我们希望贵司提供：

1. **命令行工具（CLI）**：类似 `lark-cli`，支持通过命令行完成高频文档操作。
   - 示例：`office-cli docs create --title "周报" --content "..."`
   - 示例：`office-cli sheets write --sheet-id xxx --range A1:C3 --data ...`
   - 示例：`office-cli slides create --template xxx --data ...`

2. **MCP Server（Model Context Protocol）**：如果可能，提供 MCP Server，让 LLM/Agent 可以直接通过 MCP 协议发现工具、调用 Office 能力。

3. **API 与 CLI 的能力对齐**：CLI 覆盖的能力应与 API 保持一致，避免 BOT 需要直接调用复杂 REST API。

### 4.3 BOT 独立 Token 鉴权

由于 BOT 是嵌入组织架构的独立身份，我们需要：

1. **BOT 独立应用凭证**：每个 BOT 或每类 BOT 拥有独立的 `app_id` / `app_secret` 或 token。
2. **用户身份透传**：BOT 调用 Office API 时，能够携带当前交互用户的身份，使 Office 套件按用户权限执行操作（而不是按 BOT 全局权限）。
3. **操作审计**：记录 BOT 对文档的每一次创建、读取、修改、删除操作，便于安全审计。
4. **人工确认机制**：对于敏感操作（如删除文档、修改他人文档、批量导出），支持配置“需要用户确认”或“需要管理员审批”。

**参考对标**：飞书 Aily 的权限模型——Aily 的操作权限与使用者本人严格一致，敏感操作需人工确认。

### 4.4 嵌入 IM / 组织架构

1. **文档链接在 IM 中可预览**：用户在 IM 中收到 BOT 返回的文档链接时，能够直接看到标题、摘要或缩略图。
2. **IM 消息与文档评论互通**：IM 中的讨论可以同步为文档评论，文档评论也可以推送到 IM。
3. **组织架构感知**：Office 套件能够识别 BOT 所属的组织、部门、岗位，支持按组织架构设置文档权限模板。

### 4.5 私有化部署与二次开发

1. **私有化部署**：Office 套件需要支持部署在客户自有服务器或私有云上。
2. **国产化适配**：支持国产操作系统、国产芯片、国产数据库（如客户有要求）。
3. **可二次开发**：提供清晰的 API 文档、SDK、示例代码，支持 WeLink 团队进行深度集成和品牌化改造。
4. **License 友好**：允许 WeLink 将 Office 能力作为自身产品的一部分进行商业分发（或通过采购获得该权利）。

## 5. 我们期望的集成架构

```
用户 --IM--> Mattermost BOT（数字员工）
                |
                v
        Skill Router（意图识别 + 权限检查）
                |
    +-----------+-----------+-----------+
    v           v           v           v
  Teable     Office       业务系统    LLM
  Skill    Skills/CLI     (MCP)      Service
             |
             v
     Office 套件（文档/表格/幻灯片）
```

- BOT 通过自然语言接收用户请求。
- Skill Router 识别意图并调用对应的 Office Skill。
- Office Skill 通过 CLI/API/MCP 调用 Office 套件。
- Office 套件按当前用户权限执行操作并返回结果。
- BOT 将结果以自然语言回复给用户，并返回文档链接。

## 6. 交付物与验收标准

### 6.1 厂商需提供的交付物

1. **Open API 文档**：文档、表格、幻灯片三套 API 的完整文档（含鉴权、示例、错误码）。
2. **CLI 工具包**：覆盖高频操作的命令行工具及使用文档。
3. **SDK（可选）**：Python/Node.js SDK，便于 WeLink 后端集成。
4. **部署文档**：私有化部署指南、系统要求、运维手册。
5. **License 授权文件**：明确二次开发、嵌入、商业分发的授权范围。

### 6.2 验收标准（MVP）

- [ ] BOT 能够通过 CLI/API 创建、读取、更新一份在线文档。
- [ ] BOT 能够通过 CLI/API 读取和写入表格数据。
- [ ] BOT 能够以当前用户权限执行操作，无法越权访问文档。
- [ ] BOT 对文档的操作可被审计日志记录。
- [ ] 用户在 IM 中收到的文档链接可直接打开并协作编辑。
- [ ] 提供私有化部署方案，并能在测试环境完成部署。

### 6.3 验收标准（完整版）

- [ ] 支持文档、表格、幻灯片三类 Office 能力的完整 API/CLI。
- [ ] 支持基于模板的文档/PPT 自动生成。
- [ ] 支持文档评论、权限管理、版本历史。
- [ ] 支持 MCP Server 或等效的 Agent 集成协议。
- [ ] 支持敏感操作的人工确认机制。
- [ ] 完成国产化适配（如客户要求）。
