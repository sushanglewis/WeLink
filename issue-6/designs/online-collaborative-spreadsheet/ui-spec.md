# UI Spec: 在线协同多维表格 — WeLink 壳层规格

> **阶段**:product-prototype
> **design_id**:online-collaborative-spreadsheet
> **变更**:GitHub issue #6
> **版本**:v0.1(待 PM 评审)
> **生成时间**:2026-07-15
> **范围**:WeLink 壳层 UI(导航入口 + iframe 容器 + 状态)。Teable 原生 UI 完全复用,不在本规格范围。

---

## 1. 概述

### 1.1 目标用户

| 角色 | 核心场景 |
|---|---|
| 员工/经办人 | 通过 WeLink 进入「多维表格」查看自己被分配的督办事项,在 Teable 中填报 |
| 督办专员 | 在 Teable 中维护督办主表,通过 Mattermost bot 接收升级提醒 |
| 系统管理员 | 配置 Teable 实例、SSO、workspace 映射 |

### 1.2 设计原则

1. **完全复用 Teable 原生 UI**:WeLink 只提供壳层,不重画 Teable 内部界面(决策 #1)。
2. **零登录摩擦**:SSO 嵌入后无独立登录页,用户在 WeLink 已登录态下直接进入(决策 #2)。
3. **PC 优先**:仅 PC 端,不考虑响应式/移动端(决策 #5)。
4. **状态透明**:iframe 加载、错误、空态必须给用户清晰反馈。
5. **通知可达**:bot 消息中的链接必须能直接打开 Teable 筛选视图(决策 #4)。

### 1.3 非目标(明确排除)

- 不设计 Teable 内部的表格/表单/看板/日历/画廊视图。
- 不设计督办专员 dashboard(由 Teable 原生表格承担)。
- 不设计经办人填报表单(由 Teable 原生表单视图承担)。
- 不设计字段配置界面(用户在 Teable 后台自行配置)。
- 不设计移动端界面。
- 不设计 Teable 登录页(SSO 嵌入后无独立登录)。

## 2. 目标用户流程

### 2.1 流程 A:首次进入多维表格

```
用户点击 WeLink 左侧导航「多维表格」
    → 主内容区显示加载骨架(S-02)
    → WeLink 客户端检查 SSO 会话
        ├─ 已有有效会话 → iframe 加载 Teable URL
        └─ 会话失效 → 触发 SSO 静默登录 → iframe 加载 Teable URL
    → iframe 加载完成,显示 Teable 原生 UI(S-03)
    → 用户在 Teable 中操作(切换 workspace/database/table/view)
```

### 2.2 流程 B:Mattermost bot 通知跳转

```
经办人在 Mattermost 收到 bot 私聊消息(含 Teable 筛选视图链接)
    → 点击链接
    → 唤起 WeLink 客户端(若未启动)或直接定位到主内容区
    → 主内容区切换到「多维表格」
    → iframe 加载 Teable 筛选视图 URL(带 filter 参数)
    → 经办人看到自己被分配的督办事项,开始填报
```

### 2.3 流程 C:异常处理

```
iframe 加载失败
    ├─ 网络错误 → 显示「网络异常,请检查网络后重试」+ 重试按钮(S-04a)
    ├─ SSO 失效 → 显示「登录已过期,正在重新登录」+ 自动触发 SSO(S-04b)
    ├─ Teable 服务不可用 → 显示「服务暂不可用,请稍后重试」+ 重试 + 返回入口(S-04c)
    └─ 加载超时(>15s)→ 显示「加载时间较长」+ 继续等待/取消(S-04d)
```

**「多维表格」入口规格**:

| 元素 | 规格 |
|---|---|

| 文案 | 「多维表格」,4 汉字,字号与同级一致 |

### S-02:多维表格加载态

**触发**:用户点击「多维表格」入口后,iframe 尚未完成加载。

**布局**:

```
┌─────────────────────────────────────────────────────────────────┐
│ WeLink 顶部全局栏                                                │
├──────────────┬──────────────────────────────────────────────────┤
│ 左侧导航      │              主内容区                            │
│ · 多维表格 ✓ │   ┌────────────────────────────────────┐        │
│              │   │ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │        │
│              │   │ ░░░ 加载骨架(模拟 Teable 顶部栏)   │        │
│              │   │ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │        │
│              │   │                                    │        │
│              │   │   ◌ spinner(居中)                  │        │
│              │   │   正在加载多维表格...               │        │
│              │   └────────────────────────────────────┘        │
└──────────────┴──────────────────────────────────────────────────┘
```

**规格**:
- 骨架屏模拟 Teable 顶部栏 + 左侧 Panel 结构,让用户感知即将出现的布局。
- 居中 spinner + 文案「正在加载多维表格...」。
- 加载超过 8s 显示次级提示:「加载时间较长,请稍候」。
- 加载超过 15s 自动切换到错误态(S-04d)。

### S-03:多维表格已加载态

**布局**:

```
┌─────────────────────────────────────────────────────────────────┐
│ WeLink 顶部全局栏                                                │
├──────────────┬──────────────────────────────────────────────────┤
│ 左侧导航      │ ┌────────────────────────────────────────────┐ │
│ · 多维表格 ✓ │ │ 顶部标题栏(可选):多维表格 ▾    ⛶ 全屏    │ │
│              │ ├────────────────────────────────────────────┤ │
│              │ │                                            │ │
│              │ │         Teable iframe(全屏)               │ │
│              │ │  ┌──────────┬──────────────────────────┐   │ │
│              │ │  │ Teable   │  Teable 主内容区          │   │ │
│              │ │  │ 左侧     │  (workspace/database/    │   │ │
│              │ │  │ Panel    │   table 原生 UI)          │   │ │
│              │ │  └──────────┴──────────────────────────┘   │ │
│              │ └────────────────────────────────────────────┘ │
└──────────────┴──────────────────────────────────────────────────┘
```

**规格**:
- **顶部标题栏**(可选,默认显示):
  - 标题:「多维表格」。
  - 面包屑(可选,**默认关闭**):`多维表格 > {workspace} > {database} > {table}`,通过 postMessage 从 Teable 同步;Teable 原生已有面包屑,避免重复。
  - 全屏切换:右上角图标,点击后整个客户端窗口进入全屏模式。
- **iframe**:
  - `src` 指向 Teable workspace URL。
  - `sandbox="allow-same-origin allow-scripts allow-forms allow-popups allow-downloads"`。
  - `allow="fullscreen"`。
  - 首次加载时显示 S-02 骨架;加载完成后 200ms 淡入。
- **主题同步**:WeLink 主题切换时,通过 `postMessage({type: 'theme', value: 'light|dark'})` 通知 Teable iframe。

## 5. 交互规格

### 5.1 导航切换

| 触发 | 行为 | 动画 |
|---|---|---|
| 点击「多维表格」入口 | 主内容区从当前页面切换到 S-02 → S-03 | 200ms 淡入 |
| 从「多维表格」切换到其他入口 | 保留 iframe 状态(不销毁),以便快速返回 | 200ms 淡出 |
| 再次回到「多维表格」 | 恢复 iframe 状态;若 URL 已变化(用户在 Teable 中切换过视图),保持最后 URL | 200ms 淡入 |
| 浏览器刷新 / Ctrl+R | iframe 重新加载最后 URL | 走 S-02 → S-03 |

### 5.2 bot 通知跳转

| 触发 | 行为 |
|---|---|
| 点击 bot 消息中的 Teable 链接 | ① 检查 WeLink 客户端是否运行;② 唤起并定位到「多维表格」;③ 通过 postMessage 或直接修改 iframe.src 加载筛选视图 URL |
| 筛选参数无效 | Teable 原生处理(显示空数据视图),壳层不干预 |
| 链接对应的视图无权限 | Teable 原生显示「无权限」提示,壳层不干预 |
| WeLink 客户端未启动 | 系统协议唤起(`welink://spreadsheet?view={viw_id}&filter=...`) |

### 5.3 主题切换

| 触发 | 行为 |
|---|---|
| WeLink 主题切换 | 通过 postMessage 通知 Teable iframe |
| Teable 主题切换 | 通过 postMessage 通知 WeLink 壳层(双向同步,可选) |
| 系统主题变化(仅 auto 模式) | 跟随系统主题 |

### 5.4 全屏切换

| 触发 | 行为 |
|---|---|
| 点击顶部标题栏「全屏」按钮 | WeLink 客户端窗口进入全屏 |
| 按 Esc | 退出全屏 |
| iframe 内请求全屏(Teable 原生) | 通过 `allow="fullscreen"` 授权 |

## 7. 无障碍(Accessibility)

| 项 | 要求 |
|---|---|
| 键盘可达 | 左侧导航入口、按钮、链接、全屏切换均可 Tab 聚焦,Enter 触发 |
| focus 可见 | 所有可聚焦元素必须有清晰的 focus ring(2px,主题色) |
| 屏幕阅读器 | 左侧导航入口 `<button aria-label="多维表格">`;iframe `<iframe title="多维表格">` |
| 色彩对比 | 文案/背景对比度 ≥ 4.5:1(WCAG AA) |
| 字号 | 正文字号 ≥ 14px;导航入口 ≥ 14px |
| 触控目标 | 可点击元素 ≥ 32×32px(尽管仅 PC,但便于触控屏) |
| 错误提示 | 错误态使用 `role="alert"` 自动播报 |
| 加载状态 | spinner 使用 `aria-live="polite"` + `aria-busy="true"` |
| 主题 | 明/暗主题切换后,所有状态保持对比度合规 |

## 8. 实现约束

### 8.1 技术栈

- **WeLink 客户端**:Electron / Web,主内容区使用 `<iframe>` 嵌入 Teable。
- **Teable 部署**:独立域名 `https://teable.{corp-domain}/`,与 WeLink 主域分离。
- **通信**:WeLink 壳层 ↔ Teable iframe 使用 `window.postMessage`(双向)。
- **SSO**:Teable 企业版 SSO(SAML/OIDC)或经 Keycloak/Authentik 联邦(详见 `designs/supervision-collab-va/integration-plan.md` §5)。

### 8.2 域名与 CSP

- Teable 必须配置 CSP `frame-ancestors` 允许 WeLink 域名嵌入。
- WeLink 必须配置 iframe `sandbox` 与 `allow` 属性,平衡安全与功能。

### 8.3 性能

- iframe 加载超时阈值 15s,可配置(5000–60000ms)。
- 切换导航时保留 iframe(不销毁),避免重复加载。
- 长时间未交互(如 30min)后,可显示「内容已过期」横幅,但**不强制刷新**。

### 8.4 安全

- iframe `sandbox` 必须包含 `allow-same-origin allow-scripts`,否则 SSO cookie 无法传递。
- bot 消息中的链接必须校验域名白名单,防止钓鱼链接。
- 用户敏感操作(如批量删除)由 Teable 原生二次确认承担,壳层不重复。

### 8.5 兼容

- 仅 PC 端,主流浏览器(Chrome / Edge / Safari 最新 2 个版本)。
- Electron 内嵌 Chromium 与浏览器行为一致。

5. **空态**:S-05 是否需要?还是完全交给 Teable 原生空态?
6. **snooze 行为**:「稍后提醒」的默认间隔(1h)是否合理?是否需要用户可配置?
7. **主题同步方向**:是否需要双向同步(Teable 主题变化也通知 WeLink)?默认建议单向(WeLink → Teable)。
