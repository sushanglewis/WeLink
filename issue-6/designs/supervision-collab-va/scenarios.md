# 使用场景：Teable + Mattermost 督办协作系统

> **⚠️ 选型更新(2026-07-15)**:本文档基于 Baserow 的具体实例编写,**选型已更新为 [Teable](https://github.com/teableio/teable)**。文档中的业务场景与角色交互模式仍可参考,但具体表/字段操作需按 Teable 实际能力重新映射。Baserow 仅保留作为功能完备性参照基线(见 SRS §1.3)。

## 场景 1：督办专员通过 Excel 批量新建督办事项

**角色**：督办专员、Agent（Claude）、脚本  
**入口**：Agent 对话

1. 督办专员将 Excel（蓝表） 文件以消息的形式通过IM发送给督办BOT。
2. Agent 调用脚本：`upload-excel --table-id 378 --file data.xlsx --operation append`。
3. 脚本自动化解析 Excel 并将数据批量写入 Teable 主表（table 378）。
4. 脚本返回成功导入记录数与失败明细（如必填字段缺失）、经办人匹配成功记录与失败明细。
5. Agent 向督办专员汇报汇总信息，并发送新增事项的主表筛选 URL。
6. Teable 内部公式/自动化根据主表经办人字段匹配协作者，并写入跟进表 `Collaborators`创建一条需维护三要素的记录。
7. Teable 触发 `rows.created` Webhook，延时任务服务注册 T+24h / T+36h 检查任务。

**预期结果**：督办事项成功导入，Teable 自动匹配协作者，监控任务已注册；Agent 不解析 Excel 行数据。

## 场景 2：督办专员通过任务编号批量发起历史事项跟进

**角色**：督办专员、Agent、脚本  
**入口**：Agent 对话

1. 督办专员向 Agent 发送任务编号集合，或发送Excel（绿表），任务编号合集来自于督办专员直接在excel中复制并粘贴一列
2. Agent 调用脚本：`follow-up-tasks --table-id 835 --tasks SJ20260750,SJ20260751,...`。
3. 脚本校验主表存在性，为存在的编号批量创建跟进记录。
4. 脚本返回成功创建记录与失败编号清单。
5. Agent 向督办专员汇报结果。
6. Teable 自动匹配协作者并触发 Webhook。

**预期结果**：存在的任务编号均生成跟进记录，不存在的编号返回明确失败清单。

## 场景 3：经办人接收批量填报通知

**角色**：经办人、脚本/Mattermost 通知器  
**入口**：Mattermost 私聊

1. 脚本按经办人/协作者聚合同一批次内所有待办跟进任务。
2. 脚本向每位经办人发送一条汇总通知：
   > "您有 3 项督办事项需要填报反馈，请通过链接进入查看并填写。  
   > 截止时间：2026-06-27 10:00  
   > [查看我的待办]({按协作者筛选的主表视图 URL})"
3. 经办人点击链接进入 Teable 主表筛选视图，看到所有自己被协作者的督办记录。
4. 经办人点击主表 `跟进记录表` 单元格中最新的一条记录，展开填报表单并填写。

**预期结果**：经办人一次性收到汇总通知，通过筛选视图处理全部待办事项，避免消息轰炸。

## 场景 4：24h 未填报批量提醒

**角色**：延时任务服务、经办人  
**触发**：跟进记录创建后 24 小时

1. Celery 任务 `check_follow_up_24h` 到达执行时间。
2. 服务查询对应批次内所有未填报跟进记录（7942/7943/7944 未全部非空）。
3. 服务按经办人/协作者聚合未填报记录。
4. 对每条聚合结果发送一条汇总提醒消息，包含按协作者筛选的主表 URL。
5. 对已填报的记录取消 36h 任务。

**预期结果**：未填报的跟进记录按经办人批量汇总提醒。

## 场景 5：36h 升级批量提醒

**角色**：延时任务服务、经办人、督办专员  
**触发**：跟进记录创建后 36 小时

1. Celery 任务 `check_follow_up_36h` 到达执行时间。
2. 服务再次查询跟进记录填报状态。
3. 若已填报：流程结束。
4. 若未填报：按经办人聚合后向经办人发送升级提醒，同时向督办专员发送按任务编号筛选的汇总 URL。

**预期结果**：超期 36h 未填报的事项升级到督办专员层面，且消息按人聚合避免刷屏。

## 场景 6：人工在 Teable 中直接创建跟进记录

**角色**：督办专员  
**入口**：Teable Web UI

1. 督办专员直接登录 Teable，在跟进表中手动创建一条记录。
2. Teable 触发 `rows.created` Webhook。
3. 延时任务服务接收到事件并注册 24h/36h 检查。

**预期结果**：无论通过 Agent/脚本还是人工创建，跟进记录都会进入统一的监控流程。

## 场景 7：Teable 内部协作者自动匹配

**角色**：Teable 自动化/公式  
**触发**：主表记录创建或更新

1. 主表新增一条记录，`经办人及联系方式` 字段填写标准化姓名（如"张三"）。
2. Teable 公式或自动化规则将该姓名与 Teable 用户表匹配。
3. 匹配成功后，自动在跟进表 `Collaborators` 字段写入对应 user_id。
4. 若匹配失败，该字段留空，记录本身不受影响。

**预期结果**：协作者匹配由 Teable 内部机制完成，脚本不预处理经办人映射。
