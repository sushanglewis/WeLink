# UI 设计交接文档：issue-11 企业 IM 桌面应用

## 背景与目标

基于现有 Mattermost B/S 二次开发系统，打造企业自有品牌的 Windows/macOS 桌面应用。核心目标是解决浏览器访问体验割裂、Mattermost 品牌暴露、设置不符合中国用户习惯的问题。

- **产品定位**：面向企业内部员工的 IM 桌面客户端。
- **体验对标**：飞书、钉钉等国内协同办公软件。
- **架构**：Tauri 桌面应用，混合 C/S + B/S：一级导航为 C/S 原生；聊天等核心功能初期通过 WebView 嵌入现有 B/S 页面。

## 目标用户

- **主要用户**：企业内部员工，日常办公使用 Windows PC 或 macOS。
- **次要用户**：企业 IT/管理员、产品负责人。

## 关键设计决策

| 决策项 | 结论 | 说明 |
|--------|------|------|
| 平台 | Windows 10+ / macOS 12+ | 同时支持，双平台安装包 |
| 导航形式 | **左侧边栏** | 一级导航固定在左侧，类似飞书/钉钉 |
| 内容区域 | WebView 嵌入 | 聊天、通讯录、AI 表格等功能页面初期为 WebView |
| 通知 | C/S 与 B/S 统一 | WebView 内红点/未读数需同步到侧边栏和托盘 |
| 品牌 | 完全隐藏 Mattermost | 窗口标题、图标、关于页、启动图替换为企业品牌 |
| 登录 | 应用级统一登录 | 使用已同步到 EAIC 的企业邮箱/密码登录；提供「记住我」自动登录；第三方 SSO/OAuth 后续迭代预留 |

## 必须设计的界面

### 1. 启动/登录页

- **EAIC 品牌启动页（Splash）**：展示企业 Logo + 应用名称 EAIC。
- **引导程序（Onboarding Wizard）**：
  - 步骤一：确认/输入组织 URL（预配置，允许修改）。
  - 步骤二：输入已同步到 EAIC 的企业邮箱/密码。
  - 「记住我」默认勾选，保存 refresh token 长期有效直到手动退出。
- **状态**：加载中、登录中、登录失败（输入框下方 inline 提示）。
- **约束**：不出现 Mattermost 品牌；不展示服务器 URL 输入框（或以组织首字母弱化展示）。

### 2. 主界面框架

- **左侧边栏**：
  - 顶部：企业 Logo / 应用名称 EAIC（可折叠为仅图标）。
  - 导航项：聊天、通讯录、AI 表格（图标 + 文字，含红点/未读数）。
  - 底部：用户头像、姓名、在线状态；点击弹出菜单（个人设置 / 系统设置 / 关于 / 退出登录）。
- **顶部标题栏**：完全自定义，展示应用名称、窗口控制按钮（最小化/最大化/关闭）。
- **内容区域**：WebView 容器，用于加载各功能页面；与侧边栏之间有可见分割线。
- **托盘图标**：企业品牌图标，支持右键菜单（打开、退出、设置）。

### 3. 系统通知

- **通知弹窗**：企业品牌样式，包含发送者头像、名称、消息摘要、时间。
- **托盘红点**：与 WebView 内未读数同步。
- **点击行为**：唤出应用窗口，切换到「聊天」导航，并通过 JS Bridge 定位到对应会话/频道。

### 4. 设置界面

#### 个人设置

设置弹窗采用左右分栏结构：

- **通用**：头像、全名、用户名（只读）、邮箱（只读）。
- **通知**：桌面通知开关、通知声音、邮件通知、提及词。
- **显示**：主题（浅色/深色/跟随系统，影响原生界面和 WebView 内 B/S 页面）、语言。
- **安全**：修改密码。

#### 系统设置

- **通用**：开机自启动、最小化到托盘、下载路径、语言。
- **高级**：缓存清理。
- **账号**：退出登录。

**设计原则**：命名符合中国用户习惯，避免 Mattermost 原生术语（如「Team」「Channel」等英文概念需中文化或重命名）。

### 5. 关于页面

- EAIC 品牌 About 页面。
- 版本号、企业版权信息、第三方许可声明。
- 不出现 Mattermost 版权/链接（除非法律必须保留的声明）。

## 品牌与视觉要求

- **Logo**：提供 Windows/macOS 安装包图标、任务栏图标、托盘图标（含浅色/深色模式适配）。
- **主色调**：企业提供主品牌色，用于侧边栏、按钮、选中态。
- **字体**：优先使用系统默认中文字体（Windows：微软雅黑/Segoe UI；macOS：苹方/SF Pro）。
- **圆角/阴影**：参考国内主流协同办公软件，保持现代、简洁。

## 平台差异注意

| 平台 | 特殊注意 |
|------|----------|
| Windows | 支持原生窗口边框；任务栏图标、托盘右键菜单；.exe/.msi 安装包向导界面 |
| macOS | 支持菜单栏应用菜单；Dock 图标；.dmg 安装包背景图；窗口按钮在左侧 |

## 需要 UI 同事产出的交付物

1. **低保真原型**：主界面框架、登录页、设置页流程。
2. **高保真视觉稿**：
   - 登录页
   - 主界面（含左侧边栏、WebView 内容区、托盘状态）
   - 系统通知弹窗
   - 个人设置页
   - 系统设置页
   - 关于页面
3. **切图与标注**：图标、Logo、启动图、托盘图标（1x/2x）。
4. **设计规范**：颜色、字体、间距、组件规范。

## 与开发协作要点

- 左侧边栏为 C/S 原生实现，需给出明确尺寸、折叠态、选中态、红点样式。
- WebView 内容区初期加载现有 B/S 页面，UI 需定义内容区与侧边栏的边界、加载态、错误态。
- 通知弹窗由 C/S 层触发，但内容来自 WebView，需统一视觉风格。
- 设置页若部分功能仍由 B/S 页面承载，需明确哪些设置项由原生 UI 提供，哪些由 WebView 提供。
