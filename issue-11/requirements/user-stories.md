# 用户故事: 2026-07-20-issue-11

## 用户故事一：无需输入服务器地址即可进入登录

- **作为**：企业内部员工
- **我希望**：安装并打开 EAIC 桌面应用后，无需手动输入 组织地址
- **以便**：快速进入登录流程，减少使用门槛
- **验收标准**：
  - 安装包预配置组织地址。
  - 启动后直接进入 EAIC 品牌引导程序。
  - 用户看不到组织地址 输入入口（或以组织首字母弱化展示）。

## 用户故事二：使用已同步的企业账号登录

- **作为**：企业内部员工
- **我希望**：在 EAIC 桌面应用中直接使用企业邮箱/密码登录
- **以便**：无需注册额外账号，复用企业身份体系
- **验收标准**：
  - 应用层提供统一登录入口（引导程序）。
  - 员工账号体系已同步到 EAIC，输入企业邮箱/密码即可认证。
  - 「记住我」默认勾选，保存 refresh token 长期有效。
  - 登录状态在应用生命周期内保持。

## 用户故事三：使用原生导航切换主要功能

- **作为**：企业内部员工
- **我希望**：桌面应用有原生的通讯录、聊天、AI 表格一级导航
- **以便**：获得接近飞书/钉钉的桌面应用体验
- **验收标准**：
  - 登录后展示原生一级导航。
  - 点击导航项可切换到对应功能区域。
  - 聊天区域初期通过 WebView 加载现有 B/S 二开页面。

## 用户故事四：接收新消息系统通知并定位会话

- **作为**：企业内部员工
- **我希望**：收到新消息时桌面应用弹出系统通知
- **以便**：在不主动查看应用时也能及时响应工作消息
- **验收标准**：
  - 应用在线、未处于前台聚焦、开启桌面通知时，收到新消息触发系统通知。
  - 点击通知可唤出应用窗口、切换到聊天导航并通过 JS Bridge 定位到对应会话。

## 用户故事五：最小化到托盘并快速唤出

- **作为**：企业内部员工
- **我希望**：关闭或最小化应用时它常驻系统托盘
- **以便**：不占用任务栏空间，同时能快速打开应用
- **验收标准**：
  - 关闭窗口时应用最小化到托盘而非退出。
  - 点击托盘图标可唤出应用窗口。
  - 支持开机自启动（可在设置中开关）。

## 用户故事六：隐藏 Mattermost 品牌痕迹

- **作为**：企业 IT/产品负责人
- **我希望**：桌面应用的窗口标题、图标、关于页面等不暴露 Mattermost
- **以便**：呈现企业自有 IM 产品形象
- **验收标准**：
  - 窗口标题、图标、关于页面使用 EAIC 企业品牌。
  - 不展示 Mattermost 相关外链和版权信息以外的必要声明。
  - 设置页面中移除或重命名不符合中国用户习惯的选项。

## 用户故事七：符合中国用户习惯的设置

- **作为**：企业内部员工
- **我希望**：个人设置和系统设置符合中国用户常用习惯
- **以便**：无需理解 Mattermost 原生概念即可调整常用选项
- **验收标准**：
  - 个人设置（头像、通知、主题等）布局符合国内 IM 习惯。
  - 系统设置（语言、时区、快捷键等）提供 sensible defaults。
  - 设置项命名和分组避免技术术语。
