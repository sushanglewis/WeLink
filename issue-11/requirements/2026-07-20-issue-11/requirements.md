# 需求文档

<!-- status: pending -->

## 背景

当前基于 Mattermost B/S 进行二次开发，已在外层增加一级功能目录（通信录、聊天、AI 表格），使页面视觉结构与飞书、钉钉等协同办公软件保持一致。但用户仍需要打开浏览器、输入自托管 Mattermost 地址、登录，体验割裂。此外，二开页面视觉效果较差，且 Mattermost 原有的设置、用户设置等功能与中国用户使用习惯差距较大。

（来源：GitHub Issue #11）

## 问题

1. 浏览器访问 Mattermost 的流程割裂：打开浏览器 → 输入服务器地址 → 登录。
2. 当前二开页面视觉效果不佳，细节体验不足。
3. Mattermost 默认设置、个人设置不符合中国用户习惯。
4. 缺乏独立的 PC 端应用入口，产品完整度不足。

## 目标

为现有 Mattermost B/S 二开系统打造 PC 端应用程序，实现：

- 安装包分发时预配置服务器地址，用户无需手动输入 URL。
- 支持企业邮箱 SSO 登录，减少登录门槛。
- 隐藏 Mattermost 品牌痕迹，呈现企业自有 IM 产品形象。
- 适配中国用户习惯的设置项和个人设置界面。
- 提供桌面端原生体验（任务栏/托盘、快捷键、系统通知等）。

## 用户

企业内部员工，主要使用 Windows PC 办公，部分使用 macOS。

## 方案（初定）

采用 **Web 封装方案**：使用 Tauri 构建 Windows/macOS 桌面应用，封装现有 Mattermost B/S 二开页面。

### 选型理由

- 与现有 B/S 二开页面无缝衔接，可快速复用 Web 层改造。
- 预配置服务器 URL、隐藏 Mattermost 品牌、适配中国用户习惯的成本最低。
- 无需 fork 官方 Mattermost Desktop 仓库，避免长期 upstream 合并成本。
- 许可证风险最小。

### 关键能力

- 安装包预配置服务器 URL。
- 启动后直接进入登录/SSO 页面。
- 系统级通知与托盘图标。
- 窗口管理（最小化到托盘、快捷键唤出）。
- 文件下载与打开。
- 深度链接（如从邮件/日历跳转到指定聊天）。

## 非目标

- 本次不包含移动端应用（iOS/Android），留待后续阶段。
- 不替换 Mattermost 服务端。
- 不重新实现完整的 IM 协议和消息同步机制。
- 不要求语音/视频通话等高级原生能力（若 Mattermost Web 已支持，则通过 Web 层呈现）。

## 验收标准

- [ ] Windows 安装包可正常安装并启动。
- [ ] 启动后无需用户输入服务器 URL 即可进入登录/SSO 页面。
- [ ] 企业邮箱 SSO 登录流程可正常使用。
- [ ] 应用窗口标题、图标、关于页面等不暴露 Mattermost 品牌。
- [ ] 设置和个人设置页面符合中国用户习惯（具体条目待澄清）。
- [ ] 收到新消息时可通过系统通知提醒。
- [ ] 应用可最小化到系统托盘并通过快捷键/点击唤出。

## 开放问题

1. Windows 与 macOS 的优先级？是否先 Windows 后 macOS？
2. 安装包分发方式（企业内网下载 / 可执行文件 / MSI / 应用商店）？
3. 需要隐藏/定制哪些具体的 Mattermost 设置项？
4. 是否需要系统托盘、开机自启、快捷键等桌面原生能力？
5. 是否需要离线提醒或仅在线时通知？
6. 是否需要与现有 SSO 系统做额外集成，还是复用 Web 层的邮箱 SSO？
