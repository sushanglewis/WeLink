# PRD: 2026-07-20-issue-11

## 产品目标

为现有 Mattermost B/S 二开系统打造企业自有品牌的 Windows/macOS 桌面应用，解决浏览器访问体验割裂、品牌暴露、设置不符合中国用户习惯的问题。采用混合 C/S + B/S 架构：原生一级导航框架 + WebView 嵌入现有二开页面，为后续功能 C/S 化预留扩展路径。

## 功能需求

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 安装包预配置 | 安装包内置服务器 URL，用户无需手动输入 | P0 |
| 应用级 SSO 登录 | 启动后进入应用级登录页，支持企业邮箱 SSO | P0 |
| 原生一级导航 | 通讯录、聊天、AI 表格以 C/S 导航呈现 | P0 |
| WebView 嵌入聊天 | 聊天功能区域通过 WebView 加载现有 B/S 二开页面 | P0 |
| 系统通知 | 新消息时弹出系统级通知，点击可唤出应用 | P0 |
| 系统托盘 | 关闭窗口后常驻托盘，点击托盘唤出应用 | P0 |
| 开机自启动 | 支持配置开机自启动 | P1 |
| 品牌定制 | 窗口标题、图标、关于页面、启动图替换为企业品牌 | P0 |
| 设置适配 | 个人设置和系统设置页面重排/重命名以符合中国用户习惯 | P1 |
| 文件下载/打开 | 支持聊天中文件下载并在系统默认应用中打开 | P1 |
| 深度链接 | 支持从外部链接跳转到应用内指定聊天 | P2 |
| 离线提醒 | 暂不支持离线推送；以在线时通知为主 | 非目标 |
| 移动端应用 | 本次不包含 iOS/Android | 非目标 |

## 非功能需求

- **性能**：应用启动时间 ≤ 3 秒（普通办公电脑）。
- **稳定性**：支持 7×24 小时驻留，崩溃后可选自动重启。
- **安全性**：服务器 URL 和 SSO 配置在安装包中加密或混淆，防止随意篡改。
- **可维护性**：采用 Tauri 技术栈，便于团队统一维护；保留未来功能 C/S 化的扩展接口。
- **兼容性**：同时支持 Windows 10+ 和 macOS 12+。

## 发布标准

- Windows 安装包（exe/msi）通过企业内部测试。
- macOS 安装包（dmg/pkg）通过企业内部测试。
- 登录、导航、聊天加载、通知、托盘等核心流程验收通过。
- 品牌隐藏和设置适配通过产品负责人确认。
- 文档齐全：requirements.md、user-stories.md、设计文档、部署说明。

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| WebView 嵌入 B/S 页面与原生的交互复杂度 | 中 | 定义清晰的 JS Bridge 和生命周期管理 |
| Mattermost Web 页面升级导致嵌入失效 | 中 | 锁定 Mattermost 版本，建立回归测试 |
| 企业邮箱 SSO 在桌面端适配成本 | 中 | 前期与 IT 确认 SSO 协议和回调方案 |
| macOS 签名与公证流程 | 中 | 提前准备 Apple Developer 账号和 CI 流程 |
| 后续功能 C/S 化投入超出预期 | 高 | 明确 MVP 范围，按优先级逐步迁移 |
