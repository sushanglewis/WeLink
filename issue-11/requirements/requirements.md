# 需求文档: 2026-07-20-issue-11

<!-- status: approved -->

## 背景

- Issue: #11

当前基于 Mattermost B/S 进行二次开发，已在外层增加一级功能目录（通信录、聊天、AI 表格），使页面视觉结构与飞书、钉钉等协同办公软件保持一致。但用户仍需要打开浏览器、输入组织地址、登录，体验割裂。此外，二开页面视觉效果较差，且 Mattermost 原有的设置、用户设置等功能与中国用户使用习惯差距较大。

## 问题

1. 浏览器访问 Mattermost 的流程割裂：打开浏览器 → 输入组织地址 → 登录。
2. 当前二开页面视觉效果不佳，细节体验不足。
3. Mattermost 默认设置、个人设置不符合中国用户习惯。
4. 缺乏独立的 PC 端应用入口，产品完整度不足。

## 用户

企业内部员工，使用 Windows PC 和 macOS 办公，两端需同时支持。

## 方案

采用 **混合 C/S + B/S 架构**：使用 Tauri 构建 Windows/macOS 桌面应用 EAIC，将一级功能目录（通讯录、聊天、AI 表格）提升到 C/S 层作为原生导航框架；具体功能页面继续复用现有 B/S 二开页面（通过内嵌 WebView 加载），后续可逐步将部分功能迁移为 C/S 实现。

企业员工账号体系已同步到 EAIC，用户无需注册，直接使用企业邮箱/密码登录。提供「记住我」自动登录能力。

### 关键能力

- 安装包预配置服务器 URL，Windows 和 macOS 同时支持。
- 应用级统一登录：启动后进入 EAIC 引导程序，使用已同步的企业邮箱/密码完成认证。
- 原生一级导航框架：通讯录、聊天、AI 表格以 C/S 方式呈现，支持侧边栏切换。
- 功能页面嵌入：聊天等核心功能初期通过内嵌 WebView 加载现有 B/S 二开页面。
- 系统级通知与托盘图标，点击通知定位到具体会话。
- 窗口管理（最小化到托盘、点击托盘唤出）。
- 开机自启动。
- 隐藏 Mattermost 品牌痕迹。
- 个人设置与系统设置适配中国用户习惯。

## 非目标

- 本次不包含移动端应用（iOS/Android），留待后续阶段。
- 不替换 Mattermost 服务端。
- 不要求一次性将所有功能页面 C/S 化。
- 不重新实现完整的 IM 协议和消息同步机制。
- 不要求语音/视频通话等高级原生能力。

## 验收标准

- [x] Windows 安装包（exe/msi）可正常安装并启动。
- [x] macOS 安装包（dmg/pkg）可正常安装并启动。
- [x] 启动后无需用户输入服务器 URL 即可进入应用级登录页面。
- [x] 已同步的企业邮箱/密码登录流程在应用层可正常使用。
- [x] 「记住我」默认勾选，保存 refresh token 长期有效。
- [x] 登录后展示原生一级导航：通讯录、聊天、AI 表格。
- [x] 聊天功能通过内嵌 WebView 正常加载现有 B/S 二开页面。
- [x] 应用窗口标题、图标、关于页面等不暴露 Mattermost 品牌。
- [x] 组织地址以 `lgdg.cc` 预配置并弱化为组织确认，用户无需手动输入。
- [x] 个人设置页面（头像、通知、主题等）符合中国用户习惯。
- [x] 系统设置页面（语言、开机自启、最小化到托盘、下载路径、缓存清理等）符合中国用户习惯；时区跟随系统，全局快捷键按 PRD §6.1.3 定义。
- [x] 应用在线、未处于前台聚焦、开启桌面通知时，收到新消息可通过系统通知提醒。
- [x] 点击系统通知可唤出应用、切换到聊天导航并定位到对应会话。
- [x] 应用可最小化到系统托盘并通过点击托盘唤出。
- [x] 支持开机自启动（可配置开启/关闭）。
