# 场景分析：issue-11 企业 IM 桌面应用

## 目标用户

### 主要用户

- **企业内部员工**：日常使用 Windows PC 或 macOS 办公，需要稳定、低门槛的 IM 客户端。
- **企业 IT/管理员**：负责分发、配置、维护桌面应用，关注品牌一致性和部署成本。

### 次要用户

- **企业产品负责人**：关注用户体验和品牌呈现。

## 主要场景

### 场景一：首次安装与登录

1. 员工从企业内网下载安装包并安装。
2. 打开应用，展示 EAIC 品牌启动页（Splash）。
3. 应用检查本地 token：
   - 若 token 有效，自动进入主界面。
   - 若 token 无效或不存在，进入企业品牌引导程序（Onboarding Wizard）。
4. 引导程序第一步：展示/确认组织 URL（预配置，可修改）。
5. 引导程序第二步：输入企业邮箱（用户名）和密码。员工账号体系已与 EAIC 同步，无需额外注册。
6. 勾选「记住我」（默认勾选），应用保存 refresh token，长期有效直到手动退出。
7. 登录成功后进入主界面，左侧导航显示通讯录、聊天、AI 表格。
8. 默认进入「聊天」，WebView 加载现有 B/S 聊天页面。

### 场景二：日常工作沟通

1. 员工点击托盘图标唤出应用。
2. 在聊天列表中选择会话，WebView 内完成消息收发。
3. 收到新消息时，系统托盘图标显示红点并弹出通知。
4. 点击通知直接打开应用并定位到对应会话。

### 场景三：切换功能模块

1. 员工点击左侧导航「通讯录」。
2. 主区域切换到通讯录 WebView 页面（或后续 C/S 页面）。
3. 点击「AI 表格」切换到对应功能页面。

### 场景四：个性化设置

1. 员工点击头像进入「个人设置」。
2. 在符合中国用户习惯的设置页中修改头像、通知、主题。
3. 修改「系统设置」中的语言、时区、开机自启动选项。

## 边界场景

- **网络断开**：应用显示离线提示，不强制退出；恢复网络后自动重连 WebSocket。
- **SSO 会话过期**：应用检测到 401，引导用户重新登录。
- **多开限制**：企业场景建议单实例运行，第二次启动时唤出已有窗口。
- **macOS 签名缺失**：首次安装可能出现安全提示，需配合企业证书或用户手动允许。

## 非目标场景

- 移动端使用。
- 无网络环境下的离线消息收发。
- 公开应用商店分发。
- 完全脱离 Mattermost 服务端自研 IM 后端。
