## 1. 产品概述

### 1.1 目标

为企业内部员工提供一款自有品牌的桌面 IM 客户端：

- 启动后无需输入服务器地址，token 有效时自动进入主界面。
- 支持使用已同步到 EAIC 的企业邮箱/密码登录，无需额外注册。
- 提供「记住我」自动登录能力。
- 第三方 SSO/OAuth 登录作为后续迭代能力预留。
- 左侧一级导航（通讯录 / 聊天 / AI 表格）以原生 C/S 方式呈现。
- 聊天、通讯录、AI 表格等功能页初期通过 WebView 嵌入现有 B/S 二开页面。
- 系统通知、托盘常驻、开机自启动等桌面原生体验。
- 隐藏 Mattermost 品牌痕迹，个人/系统设置适配中国用户习惯。

### 1.2 目标用户

- **主要用户**：企业内部员工，使用 Windows 10+ / macOS 12+ 办公。
- **次要用户**：企业 IT/管理员（负责分发与配置）。

### 1.3 平台

- Windows 10+（x64，安装包 `.exe` / `.msi`）
- macOS 12+（x64 + Apple Silicon，安装包 `.dmg` / `.pkg`）

## 4. 登录 / SSO 流程

### 4.1 启动与登录状态

```
启动应用
  ├─ 显示 Splash 启动页（企业 Logo）
  ├─ 检查本地 token
       ├─ 有效 → 进入主界面（默认聊天）
       └─ 无效/缺失 → 显示企业品牌引导程序
            ├─ 步骤一：组织 URL（预配置，可修改）
            ├─ 步骤二：用户名/密码 + 记住我
            └─ 登录成功 → 保存 token → 进入主界面
```

### 4.2 首次启动引导程序（Onboarding Wizard）

PM 已确认：首次安装后打开应用，用户进入引导程序。流程包含组织 URL、已同步的企业邮箱/密码输入。第三方 SSO/OAuth 授权在应用内 WebView 完成（后续迭代）。

#### 步骤一：组织

| 元素 | 说明 | 状态 |
|------|------|------|
| 页面标题 | 如「欢迎来到 WeLink」或「连接到你的组织」 | 待确认 |
| 组织首字母/Logo 展示 | 视觉上弱化 URL 输入，以组织首字母或 Logo 为主视觉 | 已确认（2026-07-21 PM） |
| 组织 URL 输入框 | 预配置服务器地址，允许用户修改 | 已确认（2026-07-21 PM） |
| 组织信息展示 | 输入/确认 URL 后，请求服务器获取组织全称与组织 Logo | 已确认 |
| 下一步按钮 | 确认组织后进入身份认证 | 待细化 |
| 错误提示 | URL 无效、无法连接服务器 | 待细化 |

#### 步骤二：身份认证

| 元素 | 说明 | 状态 |
|------|------|------|
| 用户名输入 | 企业邮箱 | 已确认（2026-07-21 PM） |
| 密码输入 | 邮箱密码 | 已确认（2026-07-21 PM） |
| SSO 登录入口 | 当前阶段不展示；用户使用已同步的企业邮箱/密码登录。第三方 SSO/OAuth 作为后续迭代预留 | 已确认（2026-07-21 PM） |
| 忘记密码链接 | 跳转企业密码重置页或邮箱系统 | 待澄清 |
| 登录按钮 | 提交认证 | 待细化 |
| 记住我 | 勾选后保存 refresh token，长期有效直到手动退出；默认勾选 | 已确认（2026-07-21 PM） |
| 错误提示 | 输入框下方 inline 提示（账号/密码错误、网络断开等） | 已确认（2026-07-21 PM） |

#### 步骤三：登录成功

- 应用获取并保存 token。
- 若用户勾选「记住我」（默认勾选），则保存 refresh token，长期有效直到用户手动退出登录。
- 若未勾选「记住我」，则 token 仅在本次会话有效，关闭应用后清除或仅保留短有效期。
- 进入主界面（默认「聊天」）。

### 4.3 第三方 SSO/OAuth 流程（后续迭代预留）

当前阶段用户使用已同步到 EAIC 的企业邮箱/密码登录。后续如需支持第三方 SSO/OAuth，流程如下：

1. 用户在引导程序中选择 SSO 登录方式。
2. 应用内 WebView 打开 SSO 授权页。
3. 用户完成企业邮箱认证。
4. 应用接收回调授权码 / token。
5. 应用交换 Mattermost token 并加密保存。
6. 进入主界面。

> **待澄清**：
> 1. 已同步的企业邮箱/密码登录与第三方 SSO/OAuth 登录是并存还是当前阶段仅保留企业邮箱/密码？（已确认：当前阶段仅企业邮箱/密码，第三方 SSO 预留）
> 2. 登录失败或网络断开时，页面如何提示？是否有「重试」按钮？（已确认：输入框下方 inline 提示）
> 3. 「记住我」默认是否勾选？token 保存策略是什么？（已确认：默认勾选，保存 refresh token 长期有效）
